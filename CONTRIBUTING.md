## CONTRIBUTING TO GCS

**ANY CONTRIBUTION CONTAINING JS OR GJS LOGIC WILL BE REJECTED WITHOUT REVIEW.**

ALL CODE MUST BE FULLY COMPATIBLE WITH .NET 10 NATIVE AOT, IF THE BUILD PRODUCED BY: `dotnet publish -c Release /p:PublishAot=true` **GENERATES ANY TRIMMING, REFLECTION, OR COMPILER WARNINGS, THE PR WILL BE DISCARDED UNTIL MANUALLY FIXED BY THE AUTHOR.** 

**NEGLIGENT, SLOPPY SUBMISSIONS ARE NOT ACCEPTED.** YOU ARE EXPECTED TO ADHERE STRICTLY TO THE `.editorconfig` AND EXECUTE **CONVENTIONAL COMMITS**.

**USE COMMON SENSE**, IF THE CODE DOES NOT WORK OR BREAKS THINGS, IT IS GARBAGE.

ASK QUESTIONS **BEFORE** COMMITTING TO A LARGE TASK. IT IS BETTER TO ASK THAN TO WASTE EVERYONE'S TIME ON A REJECTED PR. IF YOU DO THE WORK RIGHT AND FOLLOW THESE CRITERIA, NO ONE WILL SHOUT AT YOU.

BUGS **MUST** INCLUDE `dotnet --info` AND REPRODUCTION LOGS.

**THESE RULES ARE NON-NEGOTIABLE. WITHOUT THESE, A PROJECT IS JUST A CIRCUS, AND WE ARE NOT HERE TO MANAGE MONKEYS.**

**IF YOU CANNOT COMPLY WITH THESE, DO NOT SUBMIT CODE.**
